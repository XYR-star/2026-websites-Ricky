真正让我想重新做个人网站的，不是“需要一个主页”，而是需要一个可以长期维护的工作台。

一个好的个人内容站，不只是把经历堆上去。它更像一个缓慢但稳定的出版系统，让思考、研究和项目过程都能留下可回看的痕迹。

## 为什么不用 GitHub 风格

GitHub 风格当然高效，但它太像一个索引页面，不像一个有审美、有气质的出版空间。

我想要的是另一种感觉：

- 首页不是链接堆，而是一张封面
- 文章不是日志条目，而是被认真排版的内容
- 科研更新不是流水账，而是有阶段感的研究记录

## 这个站点会怎样更新

第一版选择 `Astro + Markdown`，是为了让写作和维护都尽量简单：

1. 新博客就是新增一篇 Markdown 文件。
2. 新科研进展就是新增一条研究记录。
3. 首页自动挑选最新内容，不需要手动维护很多重复信息。

这会让网站更像一个持续写作的地方，而不是一次做完就静止的作品。
