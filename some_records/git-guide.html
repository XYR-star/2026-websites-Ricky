🚀 Git & GitHub 服务器配置与使用指南

基于个人项目实战记录 | 服务器连接 GitHub 全流程

1. 核心概念：为什么要配置 SSH？

服务器连接 GitHub 就像“刷卡进门”。我们需要在服务器上生成一把“钥匙”（私钥），然后把“锁”（公钥）交给 GitHub。这样服务器以后拉取或推送代码时，就不需要每次都输入密码了。

2. 配置阶段：生成钥匙并交给 GitHub

第一步：在服务器上生成密钥

在服务器终端执行以下命令（邮箱填自己的）：

ssh-keygen -t rsa -C "你的邮箱@example.com"
注意： 执行过程中遇到提示直接按 回车 即可（不需要设置密码短语）。

第二步：获取公钥内容

生成完毕后，查看公钥内容：

cat ~/.ssh/id_rsa.pub

复制屏幕上显示的以 ssh-rsa 开头的一长串字符。

第三步：在 GitHub 上添加公钥

  1. 进入 GitHub Settings -> SSH and GPG keys。
  2. 点击 New SSH key。
  3. Title 随便填（如 Server），Key 粘贴刚才复制的内容。
  4. 点击 Add SSH key 保存。

第四步：验证连接

回到服务器，测试连接是否成功：

ssh -T git@github.com

如果看到 Hi 用户名! You've successfully authenticated... 说明配置成功！

3. 仓库阶段：创建与克隆

创建仓库

在 GitHub 网页上点击 New Repository。

  • Name: 填项目名（如 2026-websites-Ricky）。
  • Public/Private: 按需选择。
  • Add README: 建议勾选（这样仓库初始不为空，方便首次拉取）。

将仓库克隆到服务器

仓库创建好后，复制 SSH 地址，在服务器执行：

git clone git@github.com:用户名/项目名.git
cd 项目名

4. 分支管理：Main 与 Dev

现在的 GitHub 默认主分支叫 main（不再是 master）。

查看当前分支

git branch

带 * 号的就是当前所在的分支。

关于 Dev 分支

如果是个人小项目，直接在 main 分支开发即可。如果需要创建开发分支：

git checkout -b dev
git push -u origin dev

5. 日常开发：Git 操作流

Git 不会自动上传修改，需要你手动“打包”和“发货”。

标准四步走

  1. 查看状态：看看改了啥。
    git status
  2. 添加到暂存区：把修改的文件挑出来。
    git add . (点代表所有改动)
  3. 提交到本地仓库：拍照存档，写备注。
    git commit -m "这里写修改说明"
  4. 推送到 GitHub：同步到云端。
    git push
